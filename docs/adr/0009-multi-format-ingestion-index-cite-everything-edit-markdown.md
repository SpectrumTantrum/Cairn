# Multi-format ingestion: index & cite everything, edit only Markdown; Docling + whisper.cpp via a Python sidecar; per-format citation anchors

Cairn accepts a broad set of file types but makes two different promises about them: what it can **index and cite** (broad) versus what it can **edit** (Markdown only, plus PDF annotations). Non-Markdown files — PDF, DOCX, PPTX, HTML, EPUB, and audio/video — are **read-only sources**: parsed, chunked, indexed, and citable, but never agent write-targets. This is the Cursor/NotebookLM model, and it keeps the agent's write-surface small (ADR-0008). Document formats are parsed to **typed blocks via Docling (MIT)** and chunked *from those blocks*, not from a lossy Markdown export — preserving the Mneme feasibility correction and feeding the existing structure-aware chunk→embed→index path (ADR-0006). Audio and video are transcribed locally with **whisper.cpp (MIT)**; the transcript is the indexable text and citations are **timestamps that seek a player**, which brings **video into v1** (reversing the ADR-0003 / v1-scope deferral). Images and scanned PDFs go through the already-ratified OCR tier (ADR-0004). Because Docling and whisper are Python/native while the retrieval engine is TypeScript in-process (ADR-0001), ingestion/parsing runs in a **bundled Python sidecar** spawned by the Electron main process — used for parsing only, never for retrieval. Governing all of it is one load-bearing invariant: **a format ships only when citation click-through jumps to the exact source.** Every chunk carries a format-specific anchor — char offset (Markdown), page + bbox (PDF/documents), timestamp (AV) — and a format that can be indexed but not cited into does not ship.

## Considered options

- **Keep the pure-TS, no-Python stance (ADR-0001 as originally written):** simplest packaging, but there is no permissive, in-process TS equivalent of Docling for layout-aware DOCX/PPTX/EPUB parsing, and Markdown-export chunking is lossy — it drops the citation provenance (the Mneme correction). Rejected: it caps the source set at Markdown + pdf.js PDFs and forecloses the "index everything" promise.
- **Treat every accepted format as an edit target** (round-trip DOCX/PPTX/HTML/etc.): maximal, but round-tripping arbitrary formats is a large, fragile surface and multiplies the agent's write-risk (ADR-0008). Rejected — index/cite is the valuable 90%; editing stays Markdown + PDF annotations.
- **Python sidecar for ingestion (chosen):** bundle a Python runtime as a sidecar spawned by Electron's main process, used for Docling parsing + whisper transcription only. This amends ADR-0001's blanket rejection of Python and ADR-0004's "Docling rejected for v1" — but only for the ingestion path; the retrieval engine (index, embed orchestration, hybrid search) stays TS in-process, so ADR-0001's headless/CLI-testable core invariant holds.

## Consequences

- **Supersedes the video deferral.** ADR-0003's "v1 node types: `note`, `pdf`, `flashcard` — Video is deferred …" bullet and `docs/v1-scope.md`'s "Deferred → Video nodes" line are **superseded**: AV ingestion (audio + video) with timestamp citations is now **in v1**. Both locations carry a pointer back to this ADR. Audio/video *output* generation — Audio Overview / TTS (ADR-0002 Tier-2) — **remains deferred**; this ADR is only about audio/video as *input sources*.
- **Amends ADR-0001 and ADR-0004** on the "no Python runtime" corollary: a Python sidecar is now shipped, scoped to ingestion/parsing. Accepted costs: a **larger install**, **macOS codesigning of every dylib** in the embedded runtime, and keeping the runtime **outside the asar** so auto-update deltas stay small. The core retrieval engine remains pure-TS in-process — ADR-0001 is unchanged for that path.
- **Heavy models are fetched on first run, not bundled** — Docling layout/OCR weights and whisper models download on first use, consistent with the privacy stance ("zero outbound at steady state; first-run model downloads are fine"). The installer ships the runtime + code, not the model weights.
- **New permissive dependencies (license policy holds):** Docling (MIT), whisper.cpp (MIT), and an embedded CPython (PSF — permissive). No AGPL/GPL enters the ingestion path; in particular Docling reaches PDFs via pypdfium2 (BSD), avoiding the PyMuPDF trap in the standing license table.
- **Citation-anchor invariant is a per-format ship-gate.** A new format may be indexed for internal testing but must not ship to users until click-through-to-exact-source works: Markdown → char offset (ADR-0007), documents → page + bbox, AV → timestamp seek in a player. This extends ADR-0007's external-anchor model to bbox and timestamp anchors.
- **Editing stays Markdown + PDF annotations.** Every non-Markdown source is read-only to both the agent and the user-as-editor; the graph still represents AV/media as a Markdown stub + transcript beside the binary asset (ADR-0003's plain-text-face invariant), so vault portability survives.
- **Alpha guard:** this is v1 scope, not the desktop MVP alpha — `docs/mvp-scope.md` (Markdown-only) still overrides `docs/v1-scope.md` for the first GUI release; multi-format / AV ingestion does not enter the alpha.
