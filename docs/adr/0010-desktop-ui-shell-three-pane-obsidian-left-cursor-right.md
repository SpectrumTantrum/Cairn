# Desktop UI shell: three panes — Obsidian-style vault rail (left), CodeMirror 6 editor (center), Cursor-style agent sidebar (right)

Cairn's v1 desktop shell is a **three-pane layout**, ratified by Torres as "the right side agent sidebar should be like Cursor but the left side should look like Obsidian." The **left rail is Obsidian-style vault navigation** — file tree, plus backlinks/outline affordances over the vault's `[[wikilinks]]` (ADR-0003). The **center pane is a CodeMirror 6 Markdown editor** — CM6 is MIT and is the editor Obsidian itself builds on, so it is the natural permissive choice and gives us Obsidian-compatible editing behaviour for free. The **right rail is a Cursor-style agent sidebar** that narrates the agent and hosts the mode picker. Two cross-pane behaviours make the shell cohere: **agent edits render as inline accept/reject diffs in the center editor** (not in the chat panel — the sidebar only narrates), and **citation click-through opens the anchor target in the center pane** (editor for Markdown, viewer for PDF, player-seek for audio/video), tying directly into ADR-0009's per-format citation-anchor invariant. This ADR records the **v1 direction for the shell**; it is *not* the desktop MVP alpha, which stays panel-based (Search / Ask / Index) per `docs/mvp-scope.md`.

## Considered options

- **Panel-based alpha shell, kept as the v1 shape** (the current Search / Ask / Index panels): shipped and simple, but it is a headless-engine harness, not a knowledge-management surface — no editor, no vault navigation, no agent. It is deliberately the *alpha* shape (`docs/mvp-scope.md`) and was never meant to carry v1. Rejected as the v1 target; retained unchanged for the alpha.
- **Two-pane (vault tree + editor), agent as a modal/overlay:** simpler, but a modal agent fights the "narrate while you work" model and has nowhere to persist mode state, run history, or the read-vs-write trust cue. Rejected — the agent needs a persistent home.
- **Three-pane, Obsidian-left / editor-center / Cursor-right (chosen):** matches the ratified fusion identity (Obsidian vault + Cursor agent + NotebookLM grounding, CONTEXT.md) directly in the layout. Each pane maps to a product lineage the user already understands, and the left/right split cleanly separates *navigating your own knowledge* from *directing an agent over it*.

## Consequences

- **Left rail = Obsidian-style vault navigation.** File tree over the vault, with backlinks and document-outline affordances derived from `[[wikilinks]]` and headings. This is a **view over the plain-files graph** (ADR-0003), not a new store; it renders the same derived graph the index already rebuilds from wikilinks.
- **Center = CodeMirror 6 Markdown editor (MIT).** CM6 is the permissive editor Obsidian is built on, so it satisfies the license policy and gives Obsidian-compatible Markdown editing. The center pane is **polymorphic by node type**: Markdown → CM6 editor; PDF → pdf.js viewer (ADR-0004); audio/video → a player. Non-Markdown sources render read-only (ADR-0009's "index & cite everything; edit only Markdown").
- **Right rail = Cursor-style agent sidebar with a two-mode picker.** The picker is modeled on Cursor's and offers exactly two top-level modes: **Ask** (read-only, grounded, cited — the existing engine path: `search_notes` → grounded answer with citations) and **Agent** (a tool loop with write access, gated by ADR-0008's diff-preview + per-step approval + git snapshot + revert). Users choose a mode by **trust level (read-only vs write), not by task type.**
- **Specialized verbs are presets/slash-commands *within* the two modes, not sibling top-level modes.** The ratified v1 verbs — Synthesize, Recall, Plan, Explain/Tutor (v1-scope "Modes") — are surfaced as presets or slash-commands inside Ask (read verbs) or Agent (write verbs), not as a flat list of six peers. Rationale: (1) Cursor itself converged from many named modes down to Ask + Agent; (2) the read-vs-write split **degrades gracefully on weak hardware** — Ask works on the 3–4B hardware-gated always-on model (ADR-0002), while Agent can recommend BYOK/larger models for reliable tool-calling; (3) it keeps the trust boundary legible. **This verbs-as-presets mapping is the softest part of this ADR** — it can be revised (e.g. promoting a verb to a top-level mode, or regrouping presets) **without re-litigating the three-pane shell**, which is the firm decision here.
- **Agent edits render as inline accept/reject diffs in the center editor, not in the chat.** The sidebar narrates ("editing `notes/foo.md` …"); the actual change appears as **per-hunk accept/reject diffs inline in the editor**. Per-hunk approval is the trust mechanism for letting an agent write into a personal vault, and is consistent with the diff-preview + per-step approval already ratified in ADR-0008 and v1-scope. The chat panel never becomes the place where edits are reviewed.
- **Citation click-through opens the anchor target in the center pane.** A citation resolves via ADR-0009's per-format anchor and opens in the appropriate center-pane view: **Markdown → editor at the char offset/heading (ADR-0007), PDF → viewer at page + bbox flash (ADR-0004), audio/video → player seek to the timestamp (ADR-0009).** This makes the center pane the single "go to source" surface and reuses the same anchor invariant the ingestion pipeline already ship-gates on — no second citation mechanism.
- **Scope: this is post-alpha / v1, not the MVP alpha.** `docs/mvp-scope.md` (panel-based Search / Ask / Index, Markdown-only) still governs the first GUI release and is **unchanged** by this ADR. Agents must not start rebuilding the alpha UI into this shell; the three-pane shell is v1 work. `docs/v1-scope.md` and `CONTEXT.md` carry pointers to this ADR.
- **License check:** CodeMirror 6 (MIT) and pdf.js (Apache-2.0) are permissive; both are already in-policy. No new AGPL/GPL surface is introduced by the shell.
